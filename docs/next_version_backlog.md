# 次版バックログ

## 目的

`Ver.01.00` の確認中に見つかった改善候補のうち、今回のリリースには含めず、
次回以降のバージョンで計画的に対応する項目を整理する。

## 優先度 High

### 1. 長時間表示時の描画高速化

- 背景
  - 数時間規模の連続取得では、表示スパンを `30 min` 以上にした際に GUI の操作が重くなる
  - `1 hour` では実用上厳しく、`5 hours` では数秒待ちが発生した
- 今回見送る理由
  - 単純な修正ではなく、downsampling / decimation の設計が必要
  - 10 本のガス系列とヒーター系列を同時に扱うため、描画負荷の評価を伴う
- 次版での対応方針
  - 表示用データの decimation を導入する
  - `All` や長時間スパンでは再サンプリング済みデータを描画する
  - ライブ記録用データと表示用データを分離する

### 2. 定期保存とクラッシュ耐性

- 背景
  - GUI が途中で落ちた場合、CSV になるべく最新のデータが残っていてほしい
- 今回見送る理由
  - 書き込み頻度、性能、ファイル整合性のバランスを設計したい
- 次版での対応方針
  - 定期 flush または一時ファイル方式を検討する
  - 異常終了後の復旧方法もあわせて整理する

## 優先度 Medium

### 3. 横軸の時刻表現改善

- 背景
  - 現在の横軸は相対秒ベースで、時刻としての意味が直感的に分かりづらい
- 今回見送る理由
  - 絶対時刻表示、相対時間表示、切替 UI の設計を伴う
- 次版での対応方針
  - 絶対時刻と相対時間の切替を検討する
  - ラベルフォーマットとタイムゾーンの扱いも整理する

  - Windows 実行ファイル名の `bme688_logger.exe` への変更
