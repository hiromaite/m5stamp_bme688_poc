# 次版バックログ

## 目的

`Ver.02.00` の確認時点で残っている改善候補のうち、今回のリリースには含めず、
次回以降のバージョンで計画的に対応する項目を整理する。

## 優先度 High

### 1. 7 時間超の長期セッション向け表示戦略

- 背景
  - `Ver.02.00` で描画性能は大きく改善した
  - 一方で、保持方針は「直近 7 時間を完全保持」に留まっている
- 今回見送る理由
  - 現状の運用範囲では十分実用的で、優先度を一段下げられた
  - ただし 7 時間を超える長時間保持は今後の設計余地がある
- 次版での対応方針
  - 古い区間だけ粗い表示用バッファへ集約する二層構造を検討する
  - ライブ記録用データと表示用データをさらに分離する

### 2. 定期保存とクラッシュ耐性

- 背景
  - GUI が途中で落ちた場合、CSV になるべく最新のデータが残っていてほしい
- 今回見送る理由
  - 書き込み頻度、性能、ファイル整合性のバランスを設計したい
- 次版での対応方針
  - 定期 flush または一時ファイル方式を検討する
  - 異常終了後の復旧方法もあわせて整理する

## 優先度 Medium

### 3. アプリアイコン / コード署名 / 配布品質

- 背景
  - Windows 実行ファイルは PoC として成立したが、配布物としての仕上げ余地が残る
- 今回見送る理由
  - 動作成立を優先し、外観や署名までは対象外とした
- 次版での対応方針
  - アプリアイコン追加
  - 必要ならコード署名の検討
  - 配布手順の簡略化

## 参考

- `Ver.02.00` で取り込んだ主な改善
  - 長時間描画の実用性改善
  - `Relative / Clock` 時間軸
  - `Clear Plots`
  - firmware のプロファイル入力パース厳格化
  - 通常運転時の冗長ログ削減
